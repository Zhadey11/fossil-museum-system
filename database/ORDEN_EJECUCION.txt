FosilesDB — instalación desde cero (SQL Server)
===============================================

Orden obligatorio (scripts en database/):

  1. 01_base_datos.sql
  2. 02_tablas_principales.sql
  3. 03_fulltext_fosiles.sql
  4. 04_indices_vistas_sp.sql
  5. 05_datos_prueba.sql
  6. 06_solicitud_y_suscriptores.sql
  7. 07_catalogo_43_desde_imagenes.sql  — carga base de 41 fósiles únicos + multimedia por carpetas
  8. 08_enriquecimiento_41_fichas.sql   — enriquece ficha pública/investigador (taxonomía, medidas, contexto)

Reproceso opcional de catálogo:

  9. 09_reset_catalogo_seed.sql         — limpia carga 07/08 para volver a ejecutar desde cero

Post-instalación (rutas de imágenes en disco):

  cd backend && npm run apply:media-rules

  (opcional) npm run apply:media-rules:dry
  Verificación: npm run verify:fossil-media (backend)

Consultas útiles de verificación (copiar en SSMS si las necesitás):

  SELECT f.id, f.nombre, f.estado,
    (SELECT COUNT(*) FROM dbo.MULTIMEDIA m WHERE m.fosil_id = f.id AND m.deleted_at IS NULL) AS n_media
  FROM dbo.FOSIL f WHERE f.deleted_at IS NULL ORDER BY f.id;

  SELECT id, fosil_id, url, nombre_archivo FROM dbo.MULTIMEDIA WHERE deleted_at IS NULL AND (url IS NULL OR url = '');

Migraciones solo para bases antiguas (no hace falta en instalación nueva):

  database/legacy/99_migracion_renombres_cientificos.sql

Conexión backend (.env): DB_DATABASE=FosilesDB
